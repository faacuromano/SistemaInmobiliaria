## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline padding (`px-1`) |
| sm | 8px | Compact element spacing (`gap-2`, `p-2`) |
| md | 16px | Default element spacing (`gap-4`, `p-4`, `space-y-4`) |
| lg | 24px | Section padding (`gap-6`, `space-y-6`) |
| xl | 32px | Layout gaps (`gap-8`) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions: none. Existing codebase uses `space-y-3` (12px) and `space-y-4` (16px) within cards; `gap-6` (24px) for the info cards grid. New components follow these established patterns.

## Interaction Contracts

### 1. Firma Info Card (Sale Detail)

**Visibility rules:**
- HIDDEN when `sale.status` is `CONTADO` or `CESION` (exempt sales)
- VISIBLE for all other sale statuses (ACTIVA, CANCELADA, COMPLETADA)

**States:**

| State | Visual | Action |
|-------|--------|--------|
| Signing linked | Card shows: StatusBadge with signing status, formatted date, formatted time. Card is clickable (cursor-pointer). | Click opens FirmaManagementDialog |
| No signing linked | Card shows: StatusBadge "Sin firma asignada" (variant `default`), description text "Esta venta aun no tiene una firma de escritura vinculada.", two action buttons. | "Crear Nueva" opens SigningFormDialog with auto-fill. "Vincular Existente" opens FirmaManagementDialog in link mode. |

**Card layout:** Same as existing cards (Card > CardHeader with icon + title > CardContent with `space-y-3`). Icon: `PenTool` from lucide-react (already imported). Title: "Firma de Escritura".

### 2. Firma Management Dialog

**Opened by:** Clicking the Firma info card (when signing exists) or "Vincular Existente" button (when no signing).

**Layout:** Standard shadcn Dialog (`max-w-lg p-6`), matching `SigningFormDialog` width.

**Content when signing exists:**
- Signing status badge (StatusBadge)
- Date and time (formatted with `formatDate`)
- Notes (if any, `text-sm whitespace-pre-wrap text-muted-foreground`)
- Action row (flex gap-2 justify-end):
  - "Ver en Firmas" button (variant `outline`, size `sm`) -- navigates to `/firmas`
  - "Desvincular" button (variant `destructive`, size `sm`) -- opens ConfirmDialog

**Content when no signing (link mode):**
- Select/Combobox to pick from unlinked signings in the same development
- Use shadcn Select component with signing entries formatted as: `"{clientName} - {lotInfo} ({formatDate(date)} {time})"`
- Filter: only show signings where `saleId IS NULL` and `developmentId` matches current sale's development
- "Vincular" primary button (disabled until selection made)
- "Crear Nueva" secondary button (variant `outline`) -- opens SigningFormDialog with auto-fill

**Auto-fill for "Crear Nueva" from sale:**
```
clientName: `${sale.person.firstName} ${sale.person.lastName}`
lotInfo: sale.lot.block
  ? `Lote ${sale.lot.lotNumber} - Mz ${sale.lot.block} - ${sale.lot.development.name}`
  : `Lote ${sale.lot.lotNumber} - ${sale.lot.development.name}`
lotNumbers: sale.lot.lotNumber
developmentId: sale.lot.development.id
sellerId: sale.sellerId
saleId: sale.id
saleLabel: `${sale.person.firstName} ${sale.person.lastName} - Lote ${sale.lot.lotNumber}`
```

**"Desvincular" confirmation:**
- Uses ConfirmDialog component (already exists)
- Title: "Desvincular Firma"
- Description: "Se desvinculara la firma de esta venta. La firma no se eliminara, solo se quitara la relacion. Los pagos seguiran habilitados."
- Confirm button variant: `destructive`
- Action: calls `updateSigning` with `saleId: null`

### 3. Payment Blocking (Installments & Extra Charges)

**Gate determination (client-side):**
```typescript
const signingGateActive =
  sale.signingSlots &&
  sale.signingSlots.length > 0 &&
  !sale.signingSlots.some(s => s.status === "COMPLETADA");
```

This mirrors the server-side `checkSigningGate` logic:
- Exempt sales (CONTADO, CESION): Firma card hidden, `signingSlots` irrelevant, gate never active
- No signing linked: `signingSlots.length === 0`, gate NOT active (backward compat)
- Signing linked but not COMPLETADA: gate IS active

**Visual when gate active:**
- "Pagar" buttons rendered with `disabled` attribute
- Each disabled button wrapped in `TooltipProvider > Tooltip > TooltipTrigger (asChild) > Button(disabled) + TooltipContent`
- Tooltip text: "Complete la firma antes de registrar pagos"
- Tooltip uses existing shadcn Tooltip component (Radix, already installed)
- No banner, no additional warning -- disabled button + tooltip is the only feedback

**Visual when gate not active:**
- Buttons render as normal (no wrapper change needed)

### 4. Currency Equivalence in Payment Dialogs

**Placement:** Below the amount + currency field row, above the partial payment warning.

**CurrencyEquivalence component props:**
```typescript
interface CurrencyEquivalenceProps {
  enteredAmount: number;
  enteredCurrency: "USD" | "ARS";
  installmentCurrency: "USD" | "ARS";
  remainingAmount: number;
  manualRate: number | undefined;
}
```

**Exchange rate fetch strategy:**
- Fetch exchange rate on dialog mount via `fetchDolarApiRates()` (existing server action)
- Cache in component state; re-fetch not needed within single dialog session (API caches 1 hour)
- If manual rate is entered, use manual rate for equivalence instead of fetched rate

**Equivalence line:**
- Format: `"Equivale a ARS X,XXX,XXX (cotiz. $X,XXX.XX)"` or `"Equivale a USD X,XXX.XX (cotiz. $X,XXX.XX)"`
- Style: `text-sm text-muted-foreground` on a `bg-muted rounded-md p-3` container (matches existing "Monto pendiente" box)
- Shows when: entered currency differs from installment currency, or always as a cross-reference
- If exchange rate fetch fails: show `"Cotizacion no disponible — ingrese cotizacion manual"` in `text-muted-foreground`

**Coverage check (below equivalence):**

| Condition | Icon | Text | Style |
|-----------|------|------|-------|
| Amount >= remaining (covers) | `Check` (lucide) | `"Cubre la cuota pendiente (ARS X,XXX,XXX)"` | `text-emerald-700 dark:text-emerald-400` with `bg-emerald-50 dark:bg-emerald-950` container |
| Amount < remaining (short) | `AlertTriangle` (lucide) | `"No cubre la cuota completa — Faltan ARS X,XXX,XXX"` | Amber pattern: `border-amber-200 bg-amber-50 text-amber-800` (light) / `border-amber-900 bg-amber-950 text-amber-200` (dark) |

**Reactivity:** Equivalence and coverage update on every keystroke in amount field or manual rate field (via `form.watch("amount")` and `form.watch("manualRate")` -- same pattern as existing partial payment warning).

### 5. Sales List Firma Column

**Column definition:**
- Key: `"firma"`
- Label: `"Firma"`
- Position: After "Estado" column, before "Fecha" column

**Type extension needed:** Add `signingSlots` to `SaleRow` type:
```typescript
signingSlots?: { id: string; status: SigningStatus }[];
```

**Render logic:**
```
if no signingSlots or empty array:
  render "—" (em dash, text-muted-foreground)
else:
  take first signing slot
  render StatusBadge with SIGNING_STATUS_LABELS[status] and SIGNING_STATUS_COLORS[status]
```

**Badge variants for signing statuses (from constants.ts):**
| Status | Label | Badge Variant | Visual |
|--------|-------|---------------|--------|
| PENDIENTE | "Pendiente" | `default` | Primary blue background |
| CONFIRMADA | "Confirmada" | `secondary` | Muted gray background |
| COMPLETADA | "Completada" | `outline` | Outlined, no fill |
| CANCELADA | "Cancelada" | `destructive` | Red background |
| REPROGRAMADA | "Reprogramada" | `secondary` | Muted gray background |

## States & Edge Cases

| Scenario | Expected Behavior |
|----------|-------------------|
| Sale is CONTADO | Firma card hidden. Payment buttons always enabled. No Firma column badge (shows "—"). |
| Sale is CESION | Firma card hidden. Payment buttons always enabled. No Firma column badge (shows "—"). |
| Sale is ACTIVA, no signing linked | Firma card shown with "Sin firma asignada". Payment buttons ENABLED (backward compat). Firma column shows "—". |
| Sale is ACTIVA, signing PENDIENTE | Firma card shown with "Pendiente" badge. Payment buttons DISABLED with tooltip. Firma column shows "Pendiente" badge. |
| Sale is ACTIVA, signing COMPLETADA | Firma card shown with "Completada" badge. Payment buttons enabled normally. Firma column shows "Completada" badge. |
| Sale is ACTIVA, signing CANCELADA | Firma card shown with "Cancelada" badge. Payment buttons DISABLED with tooltip. Firma column shows "Cancelada" badge. |
| Exchange rate API fails | Equivalence line shows fallback text. Manual rate field still works. Coverage check still works if manual rate entered. |
| Manual rate entered | Equivalence uses manual rate instead of API rate. Equivalence text updates reactively. |
| Amount is 0 or empty | Equivalence and coverage check hidden (no useful info to show). |
| Multi-signing on same sale | Take first `signingSlots[0]` for display (array ordered by date desc from Phase 8 query). |

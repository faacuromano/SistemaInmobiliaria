**`describe('ACT-05: payExtraCharge — recalculation triggered after full payment')`:**

- `it('calls recalculateInstallments with saleId and amount after fully-paid extra charge')`:
  - Mock `prismaMock.extraCharge.findUnique.mockResolvedValue(buildMockExtraCharge({ amount: 5000, paidAmount: 0, status: 'PENDIENTE' }))`.
  - Mock `prismaMock.cashMovement.create.mockResolvedValue({ id: 'cm-1' } as any)`.
  - Mock `prismaMock.extraCharge.update.mockResolvedValue({} as any)`.
  - Call `payExtraCharge({ success: false, error: '' }, buildExtraChargePaymentFormData({ amount: '5000' }))`.
  - Assert `{ success: true }`.
  - Assert `vi.mocked(recalculateInstallments)` was called with `('sale-1', 5000)` — proves the chain is wired with correct args.

- `it('does NOT call recalculateInstallments on partial payment (PARCIAL)')`:
  - Mock extra charge with `amount: 5000, paidAmount: 0`.
  - Call with `buildExtraChargePaymentFormData({ amount: '2000' })` — partial, 2000 < 5000.
  - Mock `prismaMock.extraCharge.update.mockResolvedValue({} as any)`.
  - Assert `{ success: true }`.
  - Assert `prismaMock.extraCharge.update` called with `data: expect.objectContaining({ status: 'PARCIAL' })`.
  - Assert `vi.mocked(recalculateInstallments)` was NOT called — the becamePaid flag should be false.

- `it('calls recalculateInstallments when payment completes a partially-paid extra charge')`:
  - Mock extra charge with `amount: 5000, paidAmount: 3000, status: 'PARCIAL'` (remaining = 2000).
  - Call with `buildExtraChargePaymentFormData({ amount: '2000' })`.
  - Assert `{ success: true }`.
  - Assert `vi.mocked(recalculateInstallments)` was called with `('sale-1', 5000)` — the amount arg is `Number(extraCharge.amount)` i.e. the TOTAL charge amount, not the payment amount.

**`describe('ACT-06: payExtraCharge — partial failure when recalculation throws')`:**

- `it('returns error but payment data is committed when recalculation throws')`:
  - Mock extra charge with `amount: 5000, paidAmount: 0, status: 'PENDIENTE'` — will become fully paid.
  - Mock `prismaMock.cashMovement.create.mockResolvedValue({ id: 'cm-1' } as any)`.
  - Mock `prismaMock.extraCharge.update.mockResolvedValue({} as any)`.
  - Configure `vi.mocked(recalculateInstallments).mockRejectedValue(new Error('DB connection lost'))`.
  - Call `payExtraCharge({ success: false, error: '' }, buildExtraChargePaymentFormData({ amount: '5000' }))`.
  - Assert result `{ success: false, error: 'Error al procesar el pago' }` — the outer catch returns this.
  - Assert `prismaMock.cashMovement.create` was called (1 time) — proves the transaction DID execute.
  - Assert `prismaMock.extraCharge.update` was called with `data: expect.objectContaining({ status: 'PAGADA' })` — proves the update was committed.
  - Assert `vi.mocked(recalculateInstallments)` was called with `('sale-1', 5000)` — proves the chain was wired even though it threw.

- `it('rejects payment with missing extraChargeId')`:
  - Call with `buildExtraChargePaymentFormData({ extraChargeId: '' })`.
  - Assert `{ success: false, error: 'ID de cargo extra requerido' }`.

This test is the critical proof that payment data persists (committed inside `$transaction`) even when the post-transaction `recalculateInstallments` call fails. The mock structure makes this possible because `$transaction` mock resolves successfully (running the callback), then `recalculateInstallments` mock rejects.

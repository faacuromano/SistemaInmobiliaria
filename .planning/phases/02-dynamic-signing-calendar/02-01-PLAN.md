# Plan 02-01: Utility Functions and Server Data Flow

## Objective

Create pure utility functions for generating calendar segments and enabled-day dates from `BusinessHoursConfig`, expand the `getSigningsByWeek` query range to cover all 7 days, and wire `businessHours` data from the server page into the view component. This provides the complete data layer that Plan 02 will consume for dynamic UI rendering.

## Context

Phase 1 delivered `BusinessHoursConfig` type, `getBusinessHours()` server action, and the admin settings UI. The signing calendar currently uses hardcoded `TIME_SLOTS` (10 entries), `DAY_LABELS` (Mon-Fri), and a static morning/afternoon split. This plan builds the computation layer that translates the stored config into calendar-ready data structures.

**Key files from Phase 1:**
- `src/lib/business-hours.ts` -- Has `BusinessHoursConfig`, `BreakPeriod`, `DEFAULT_BUSINESS_HOURS`
- `src/server/actions/business-hours.actions.ts` -- Has `getBusinessHours()` returning `BusinessHoursConfig`

**Files to modify:**
- `src/lib/business-hours.ts` -- Add `CalendarSegment` types + `generateCalendarSegments()` + `getEnabledWeekDates()`
- `src/server/actions/signing.actions.ts` -- Expand `getSigningsByWeek()` date range to Mon-Sun
- `src/app/(dashboard)/firmas/page.tsx` -- Fetch `businessHours`, pass to `SigningsView`

**Research patterns to follow:**
- Pattern 1 (Time Slot Generation): `CalendarSlotGroup` / `CalendarBreakGroup` discriminated union
- Pattern 2 (Dynamic Day Columns): `getEnabledWeekDates()` with Sunday offset formula `dayOfWeek === 0 ? 6 : dayOfWeek - 1`
- Pattern 5 (Adaptive Week Navigation): Query full Mon-Sun range to avoid waterfall dependency on `getBusinessHours()`

**Anti-patterns from research:**
- Do NOT filter out-of-hours data at the query level (DCAL-04 requires visibility)
- Keep Monday as the week anchor -- do not change `getMonday()` logic

<task type="auto">
  <name>Add calendar segment utilities to business-hours.ts</name>
  <files>src/lib/business-hours.ts</files>
  <action>
    Add the following exports to the existing `src/lib/business-hours.ts` file (which already has `BusinessHoursConfig`, `BreakPeriod`, `DEFAULT_BUSINESS_HOURS`):

    1. **Types** -- Add discriminated union types:
       - `CalendarSlotGroup`: `{ type: "slots"; slots: string[] }` -- a group of 30-min time slots
       - `CalendarBreakGroup`: `{ type: "break"; label: string; startTime: string; endTime: string }` -- a break separator
       - `CalendarSegment`: `CalendarSlotGroup | CalendarBreakGroup` -- the union type

    2. **`generateCalendarSegments(config: BusinessHoursConfig): CalendarSegment[]`** -- Pure function:
       - Sort breaks by `startTime` (ascending string comparison -- works because HH:MM is zero-padded)
       - Walk from `openingTime` to `closingTime`, inserting break separators between slot groups
       - For each break: generate slots from cursor to break.startTime, push break segment, advance cursor to break.endTime
       - After last break: generate remaining slots from cursor to closingTime
       - Filter out empty slot groups (when break starts exactly at cursor)
       - Internal helper `generateSlotRange(from: string, to: string): string[]` generates 30-min intervals: parse HH:MM, increment by 30 min, pad with leading zeros, stop when reaching `to`

       Example: config `09:00-17:00` with lunch `12:00-14:00` produces:
       ```
       [
         { type: "slots", slots: ["09:00","09:30","10:00","10:30","11:00","11:30"] },
         { type: "break", label: "Mediodia", startTime: "12:00", endTime: "14:00" },
         { type: "slots", slots: ["14:00","14:30","15:00","15:30","16:00","16:30"] }
       ]
       ```

    3. **`DAY_NAMES` constant** -- Map of day-of-week number to Spanish label:
       `{ 0: "Domingo", 1: "Lunes", 2: "Martes", 3: "Miercoles", 4: "Jueves", 5: "Viernes", 6: "Sabado" }`
       (no accents -- matches existing calendar style in the codebase)

    4. **`getEnabledWeekDates(weekStart: Date, enabledDays: number[]): { date: Date; dayOfWeek: number; label: string }[]`** -- Pure function:
       - Sort `enabledDays` ascending
       - For each enabled day, compute offset from Monday: `dayOfWeek === 0 ? 6 : dayOfWeek - 1`
       - Create a new Date at `weekStart + offset` days
       - Return array of `{ date, dayOfWeek, label: DAY_NAMES[dayOfWeek] }`
       - `weekStart` is always a Monday (the existing `getMonday()` functions guarantee this)

    5. **`getAllSlotTimes(segments: CalendarSegment[]): Set<string>`** -- Helper for out-of-hours detection:
       - Flatmap all segments of type "slots" to get their slot arrays
       - Return as a `Set<string>` for O(1) lookup

    IMPORTANT: Keep all existing exports (`BreakPeriod`, `BusinessHoursConfig`, `DEFAULT_BUSINESS_HOURS`) unchanged. Only ADD new code.

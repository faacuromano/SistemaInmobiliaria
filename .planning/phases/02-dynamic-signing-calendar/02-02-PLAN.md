# Plan 02-02: Dynamic Calendar UI Rendering

## Objective

Replace all hardcoded calendar values (TIME_SLOTS, DAY_LABELS, morning/afternoon split) with dynamic rendering driven by `BusinessHoursConfig`. Add out-of-hours signing visibility and adapt week navigation to configured days. After this plan, the signing calendar fully reflects the admin's business hours settings.

## Context

Plan 01 delivered:
- `generateCalendarSegments(config)` -- produces `CalendarSegment[]` (alternating slot groups and break separators)
- `getEnabledWeekDates(weekStart, enabledDays)` -- produces `{ date, dayOfWeek, label }[]` for enabled days
- `getAllSlotTimes(segments)` -- produces `Set<string>` for out-of-hours detection
- `DAY_NAMES` -- Spanish day name lookup
- `businessHours` prop flowing from `page.tsx` -> `SigningsView`
- `getSigningsByWeek` now queries full Mon-Sun range

**Current state of calendar (to be replaced):**
- `TIME_SLOTS`: hardcoded 10-entry array `["09:00", "09:30", ..., "16:30"]`
- `DAY_LABELS`: hardcoded `["Lunes", "Martes", "Miercoles", "Jueves", "Viernes"]`
- `getWeekDates()`: generates 5 dates (Mon-Fri) from weekStart
- `morningSlots`/`afternoonSlots`: filter by `< "12:00"` / `>= "12:00"`
- Grid: `grid-cols-[80px_repeat(5,1fr)]` (hardcoded 5 columns)
- Separator: single hardcoded "MEDIOD." row between morning/afternoon
- `formatWeekRange()` in signings-view.tsx: computes Mon-Fri range

**Research patterns to follow:**
- Pattern 1: Segment-based rendering (iterate `segments`, render `CalendarRow` for slots, break row for breaks)
- Pattern 2: Dynamic grid via inline `style={{ gridTemplateColumns: \`80px repeat(\${dayCount}, 1fr)\` }}`
- Pattern 3: Break separator rows with label + time range
- Pattern 4: Out-of-hours detection via Set membership, rendered as extra rows with amber/dashed styling
- Pattern 5: `formatWeekRange()` adapted to use first/last enabled day

**Anti-patterns from research:**
- Do NOT use Tailwind dynamic class interpolation for grid columns -- use inline style
- Do NOT compute segments without `useMemo` -- wrap with config as dependency
- Do NOT hide out-of-hours signings -- always display them with indicator
- Keep Monday as week anchor -- Sunday is offset 6 from Monday

    ```typescript
    function formatWeekRange(monday: Date, enabledDays: number[]): string {
      const sorted = [...enabledDays].sort((a, b) => a - b);
      const firstDayOfWeek = sorted[0];
      const lastDayOfWeek = sorted[sorted.length - 1];

      const firstOffset = firstDayOfWeek === 0 ? 6 : firstDayOfWeek - 1;
      const lastOffset = lastDayOfWeek === 0 ? 6 : lastDayOfWeek - 1;

      const firstDate = new Date(monday);
      firstDate.setDate(monday.getDate() + firstOffset);

      const lastDate = new Date(monday);
      lastDate.setDate(monday.getDate() + lastOffset);

      const months = [
        "Enero", "Febrero", "Marzo", "Abril", "Mayo", "Junio",
        "Julio", "Agosto", "Septiembre", "Octubre", "Noviembre", "Diciembre",
      ];

      const fDay = firstDate.getDate();
      const lDay = lastDate.getDate();

      if (firstDate.getMonth() === lastDate.getMonth()) {
        return `${fDay} - ${lDay} de ${months[firstDate.getMonth()]} ${firstDate.getFullYear()}`;
      }

      if (firstDate.getFullYear() === lastDate.getFullYear()) {
        return `${fDay} ${months[firstDate.getMonth()]} - ${lDay} ${months[lastDate.getMonth()]} ${firstDate.getFullYear()}`;
      }

      return `${fDay} ${months[firstDate.getMonth()]} ${firstDate.getFullYear()} - ${lDay} ${months[lastDate.getMonth()]} ${lastDate.getFullYear()}`;
    }
    ```

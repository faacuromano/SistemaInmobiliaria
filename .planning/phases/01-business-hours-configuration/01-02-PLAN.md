```typescript
const DAY_OPTIONS = [

```

3. **State management:**
   - `useActionState` wrapping `updateBusinessHours` with initial state `null` (same pattern as SystemConfigSection)
   - `useForm<BusinessHoursInput>` with `zodResolver(businessHoursSchema)` and `defaultValues` from `config` prop
   - `useFieldArray({ control: form.control, name: "breaks" })` for dynamic breaks

4. **useEffect for toast** (identical pattern to SystemConfigSection):
   ```typescript
   useEffect(() => {
     if (!state) return;
     if (state.success) {
       toast.success("Horarios guardados");
       router.refresh();
     } else {
       toast.error(state.error);
     }
   }, [state, router]);
   ```

5. **Form submission approach** -- use the simpler direct-call pattern from the research:
   ```typescript
   const handleSave = form.handleSubmit(async (values) => {
     const formData = new FormData();
     formData.set("business_hours_json", JSON.stringify(values));
     // Trigger the action through a ref or direct call
   });
   ```

   **Implementation detail:** Use a hidden input + form action approach for consistency with useActionState:
   - Keep a `hiddenJsonRef = useRef<HTMLInputElement>(null)`
   - On form `onSubmit` via `form.handleSubmit`, serialize values to JSON, set the hidden input's value, then call `formRef.current?.requestSubmit()` with a slight delay (setTimeout 0) to allow React state to flush.

   **Alternatively (PREFERRED -- simpler):** Skip `useActionState` and call the server action directly from `handleSubmit`. This avoids the hidden input complexity. Use local `isPending` state:
   ```typescript
   const [isPending, setIsPending] = useState(false);

   const onSubmit = form.handleSubmit(async (values) => {
     setIsPending(true);
     try {
       const formData = new FormData();
       formData.set("business_hours_json", JSON.stringify(values));
       const result = await updateBusinessHours(null, formData);
       if (result.success) {
         toast.success("Horarios guardados");
         router.refresh();
       } else {
         toast.error(result.error);
       }
     } catch {
       toast.error("Error al guardar los horarios");
     } finally {
       setIsPending(false);
     }
   });
   ```
   Use a `<form onSubmit={onSubmit}>` (regular form, not server action form). This is cleaner because the data is structured JSON, not flat FormData.

6. **JSX layout -- three Cards inside a single `<Form {...form}><form onSubmit={onSubmit}>`:**

   **Card 1: "Horario Laboral"** (opening + closing time)
   - Two `<Input type="time">` fields in a `grid grid-cols-2 gap-4` layout
   - FormField for `openingTime` with label "Hora de apertura"
   - FormField for `closingTime` with label "Hora de cierre"
   - Each uses `<Input type="time" {...field} />` inside FormControl

   **Card 2: "Descansos"** (dynamic breaks list)
   - Map over `fields` from useFieldArray. For each break (keyed by `field.id`):
     - Row layout: `flex items-end gap-2`
     - Three FormFields: `breaks.${index}.label` (Input, flex-1, placeholder "Almuerzo"), `breaks.${index}.startTime` (Input type="time"), `breaks.${index}.endTime` (Input type="time")
     - Trash2 icon button: `<Button type="button" variant="ghost" size="icon" onClick={() => remove(index)}><Trash2 className="h-4 w-4 text-destructive" /></Button>`
   - Below the list: `<Button type="button" variant="outline" size="sm" onClick={() => append({ label: "", startTime: "", endTime: "" })}><Plus className="h-4 w-4 mr-1" /> Agregar descanso</Button>`
   - Display root-level `breaks` error from `form.formState.errors.breaks?.root?.message` or `form.formState.errors.breaks?.message` as a `<p className="text-sm text-destructive">` below the break list (this is where the cross-field refinement errors for overlapping/out-of-range show up)

   **Card 3: "Dias Habilitados"** (day toggles)
   - Map over `DAY_OPTIONS`. For each day:
     - `flex items-center justify-between` row with `<Label>{day.label}</Label>` on the left and `<Switch>` on the right
     - Switch `checked={enabledDays.includes(day.value)}` where `enabledDays = form.watch("enabledDays")`
     - Switch `onCheckedChange`: if checked, add `day.value` to array and sort; if unchecked, filter it out. Use `form.setValue("enabledDays", newArray, { shouldValidate: true })`
   - Display `enabledDays` error from `form.formState.errors.enabledDays?.message` as `<p className="text-sm text-destructive">`

   **Submit button:** `<div className="flex justify-end"><Button type="submit" disabled={isPending}>{isPending ? "Guardando..." : "Guardar Horarios"}</Button></div>`

**Important anti-patterns to AVOID:**
- Do NOT use `index` as React key for break rows -- use `field.id` from useFieldArray
- Do NOT use `useActionState` with `action={formAction}` -- use the direct call approach since the data is structured JSON
- Do NOT forget to import `useState` (needed for `isPending`)

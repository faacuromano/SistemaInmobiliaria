  <action>
Create two files:

**File 1: `src/lib/business-hours.ts`**

Define and export:
- `BreakPeriod` interface: `{ label: string; startTime: string; endTime: string }` (HH:MM format strings)
- `BusinessHoursConfig` interface: `{ openingTime: string; closingTime: string; breaks: BreakPeriod[]; enabledDays: number[] }` where enabledDays uses JS day-of-week convention (0=Sun, 1=Mon, ..., 6=Sat)
- `DEFAULT_BUSINESS_HOURS` constant of type `BusinessHoursConfig` with values: openingTime "09:00", closingTime "17:00", breaks array with one entry `{ label: "Mediodia", startTime: "12:00", endTime: "14:00" }`, enabledDays `[1, 2, 3, 4, 5]` (Mon-Fri)

**File 2: `src/schemas/business-hours.schema.ts`**

Import `z` from "zod". Build the schema in layers:

1. `timeRegex` = `/^([01]\d|2[0-3]):[0-5]\d$/` -- enforces zero-padded 24-hour HH:MM format
2. `timeStringSchema` = `z.string().regex(timeRegex, "Formato HH:MM requerido")`
3. `breakPeriodSchema` = `z.object({ label: z.string().min(1, "Nombre requerido").max(50), startTime: timeStringSchema, endTime: timeStringSchema }).refine((b) => b.startTime < b.endTime, { message: "El inicio del descanso debe ser anterior al fin", path: ["endTime"] })`
4. `businessHoursSchema` = `z.object({ openingTime: timeStringSchema, closingTime: timeStringSchema, breaks: z.array(breakPeriodSchema).max(10, "Maximo 10 descansos"), enabledDays: z.array(z.number().int().min(0).max(6)).min(1, "Al menos un dia debe estar habilitado") })`

Add three `.refine()` calls chained on `businessHoursSchema`:
- Refinement 1: `data.openingTime < data.closingTime` -- message: "La hora de apertura debe ser anterior a la de cierre", path: ["closingTime"]
- Refinement 2: `data.breaks.every(b => b.startTime >= data.openingTime && b.endTime <= data.closingTime)` -- message: "Los descansos deben estar dentro del horario laboral", path: ["breaks"]
- Refinement 3: Sort breaks by startTime, check `sorted[i].endTime <= sorted[i+1].startTime` for all consecutive pairs -- message: "Los descansos no pueden superponerse", path: ["breaks"]

Export: `businessHoursSchema` and `type BusinessHoursInput = z.infer<typeof businessHoursSchema>`.

**Important:** String comparison for HH:MM works correctly ONLY because the regex enforces zero-padded format. This is by design -- do NOT convert to minutes for comparison.

<task type="auto">
  <name>Task 2: Create server actions for reading and writing business hours</name>

  <action>
Create `src/server/actions/business-hours.actions.ts` with the `"use server"` directive.

**Imports:**
- `revalidatePath` from `next/cache`
- `requirePermission` from `@/lib/auth-guard`
- `systemConfigModel` from `@/server/models/system-config.model`
- `businessHoursSchema` from `@/schemas/business-hours.schema`
- `DEFAULT_BUSINESS_HOURS` and `type BusinessHoursConfig` from `@/lib/business-hours`
- `type ActionResult` from `@/types/actions`

**Constants:**
- `CONFIG_KEY = "business_hours"` (the SystemConfig key)

**Function 1: `getBusinessHours(): Promise<BusinessHoursConfig>`**
1. Call `systemConfigModel.get(CONFIG_KEY)`
2. If null, return `DEFAULT_BUSINESS_HOURS` (fresh install case -- BHRS-07)
3. Try `JSON.parse(raw)`, then `businessHoursSchema.safeParse(parsed)`
4. If parse succeeds, return `result.data`
5. If safeParse fails (partial/corrupted data), return `{ ...DEFAULT_BUSINESS_HOURS, ...parsed }` as a best-effort merge
6. If JSON.parse throws, return `DEFAULT_BUSINESS_HOURS`
7. Wrap steps 3-6 in try/catch

**Function 2: `updateBusinessHours(_prevState: ActionResult | null, formData: FormData): Promise<ActionResult>`**
1. Call `await requirePermission("config:manage")` for authorization
2. Extract `formData.get("business_hours_json")` -- if not a string, return `{ success: false, error: "Datos invalidos" }`
3. `JSON.parse(jsonStr)` inside try/catch
4. `businessHoursSchema.safeParse(data)` -- if fails, return `{ success: false, error: parsed.error.errors[0].message }`
5. On success: `await systemConfigModel.set(CONFIG_KEY, JSON.stringify(parsed.data))`
6. Call `revalidatePath("/configuracion")` and `revalidatePath("/firmas")` (pre-invalidate for Phase 2)
7. Return `{ success: true }`
8. Catch block: return `{ success: false, error: "Error al procesar la configuracion" }`

**Pattern note:** Follow the exact same pattern as `system-config.actions.ts` -- `"use server"` directive at top, `requirePermission` for auth, Zod `safeParse` for validation, `systemConfigModel` for persistence.

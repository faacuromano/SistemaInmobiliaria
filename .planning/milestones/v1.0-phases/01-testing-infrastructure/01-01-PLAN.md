<task type="auto">
  <name>Task 1: Install Vitest dependencies and create configuration</name>
  <files>vitest.config.ts, package.json</files>
  <action>
1. Install dev dependencies:
```bash
npm install -D vitest @vitejs/plugin-react jsdom @vitest/coverage-v8 vite-tsconfig-paths vitest-mock-extended
```

2. Create `vitest.config.ts` at project root with this exact content:
```typescript
import { defineConfig } from 'vitest/config'
import react from '@vitejs/plugin-react'
import tsconfigPaths from 'vite-tsconfig-paths'

export default defineConfig({
  plugins: [tsconfigPaths(), react()],
  test: {
    environment: 'jsdom',
    include: ['src/**/__tests__/**/*.test.{ts,tsx}', 'src/**/*.test.{ts,tsx}'],
    coverage: {
      provider: 'v8',
      include: ['src/lib/**', 'src/server/**', 'src/schemas/**'],
      exclude: ['src/generated/**', 'src/**/__tests__/**'],
    },
  },
})
```

Key decisions:
- `tsconfigPaths()` BEFORE `react()` in plugins array (path resolution must happen first).
- `include` pattern discovers both `__tests__/` directory tests and co-located `*.test.ts` files.
- Coverage scoped to business logic directories only. `src/generated/**` excluded (Prisma generated client). `src/**/__tests__/**` excluded (test helpers).
- Do NOT set `coverage.all` (removed in Vitest 4).
- jsdom environment set globally (safe default for this project).

3. Add three scripts to `package.json` `scripts` section:
```json
"test": "vitest",
"test:run": "vitest run",
"test:coverage": "vitest run --coverage"
```

- `npm test` runs Vitest in watch mode (interactive dev). On CI (non-TTY), it runs once and exits.
- `npm run test:run` always runs once and exits.
- `npm run test:coverage` runs once with V8 coverage report.
  </action>

  </verify>
  <done>Vitest runs without crashing. `npm run test:run` exits cleanly (may show "no test files" if smoke test not yet created — that is acceptable at this step).</done>

```typescript
import { describe, it, expect } from 'vitest'

describe('test infrastructure', () => {
  it('runs a basic assertion', () => {
    expect(1 + 1).toBe(2)
  })

  it('resolves @/ path aliases', async () => {
    const { MONTH_NAMES } = await import('@/lib/sale-helpers')
    expect(MONTH_NAMES).toHaveLength(12)
    expect(MONTH_NAMES[0]).toBe('ENERO')
  })
})
```

This test verifies two things:
- Vitest executes and assertions work (basic sanity).
- `vite-tsconfig-paths` correctly resolves `@/lib/sale-helpers` to `src/lib/sale-helpers.ts` (path alias verification).

Do NOT import any module that transitively imports `@/lib/prisma` — that would attempt a real database connection. `sale-helpers.ts` is a pure module with no Prisma dependency.

3. Run `npm run test:run` and verify both tests pass.

4. Run `npm run test:coverage` and verify coverage report is generated without errors. The coverage will be near 0% because we're only importing `sale-helpers` — that's expected. The goal is proving the coverage toolchain works.

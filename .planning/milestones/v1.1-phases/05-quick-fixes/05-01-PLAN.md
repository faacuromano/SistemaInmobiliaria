**Part B — Fix status reset on sold lots (DEV-02):**

The root cause: When `disabled={!isStatusEditable}` is set on the Radix `<Select>`, its internal hidden `<select>` element is also disabled and browsers do not include disabled form elements in FormData submissions. So `formData.get("status")` returns `null` on the server. The Zod schema `lotCreateSchema` has `.default(LotStatus.DISPONIBLE)`, which converts the missing status to DISPONIBLE — silently corrupting the data.

Fix: Add a hidden input that always submits the status value. When the Select is disabled (non-editable status), this hidden input ensures the current status reaches the server:

```tsx
{/* Right before the status FormField, add: */}
{!isStatusEditable && (
  <input type="hidden" name="status" value={defaultValues!.status} />
)}
```

This hidden input uses `name="status"` which will be picked up by `formData.get("status")` in the server action. When `isStatusEditable` is true, the Select component itself submits the value normally (it has `name="status"` and is not disabled). When `isStatusEditable` is false, the disabled Select does not submit, but the hidden input does.

Also, remove the `name="status"` prop from the `<Select>` component to avoid duplicate name attributes when the Select IS enabled. Instead, the react-hook-form binding with `field.onChange` + `field.value` already manages the controlled state. To ensure the value is always submitted, add a hidden input that reads from the form state for ALL cases:

Actually, the simplest and most robust approach: Always render a hidden input for status, remove `name` from Select:

```tsx
{/* Always submit status via hidden input */}
<input type="hidden" name="status" value={form.watch("status")} />

<FormField
  control={form.control}
  name="status"
  render={({ field }) => (
    <FormItem>
      <FormLabel>Estado</FormLabel>
      <Select
        value={field.value}
        onValueChange={field.onChange}
        disabled={!isStatusEditable}
      >
        {/* Remove name="status" from Select — hidden input handles submission */}
        ...
      </Select>
      <FormMessage />
    </FormItem>
  )}
/>
```

This way:
- The hidden input always submits the correct status value
- When the Select is enabled, user changes update the form state via `field.onChange`, which updates `form.watch("status")`, which updates the hidden input
- When the Select is disabled, the form state retains `defaultValues.status`, the hidden input submits it
- No risk of `null` reaching the server

Run `npx tsc --noEmit` after changes to verify no type errors.
